# 1ux4ry Blog 使用说明

这份文档是给 `Z:\my_blogs` 这个 Hugo 博客项目准备的日常维护说明。

你以后主要会做 5 件事：

1. 本地预览博客
2. 修改文章、首页和样式
3. 提交源码到 `main`
4. 一键发布静态页面到 `master`
5. 检查 `luxuryzcx.github.io` 是否正常上线

---

## 1. 项目位置

项目目录：

```text
Z:\my_blogs
```

以后所有命令，默认都先进入这里：

```powershell
cd "Z:\my_blogs"
```

---

## 2. 项目结构说明

### `content/`

放博客内容。

重点目录：

```text
content\posts\
```

这里每个 Markdown 文件基本就是一篇文章。

### `layouts/`

放 Hugo 模板。

你这个项目最常改的是：

```text
layouts\partials\profile-hero.html
```

这个文件控制首页最上面的终端风首屏。

### `static/`

放静态资源。

重点：

```text
static\css\main.css
static\js\main.js
static\images\
```

分别对应：

- 全站样式
- 首页终端动画
- 图片资源

### `data/`

放结构化数据。

你这里主要是：

```text
data\friends.toml
```

用于生成友链数据。

### `config.toml`

站点核心配置文件。

这里可以改：

- `baseURL`
- 站点标题
- 作者名
- GitHub 链接
- 邮箱
- 顶部菜单

## 3. 本地预览

启动本地预览：

```powershell
cd "Z:\my_blogs"
hugo server --bind 127.0.0.1 --port 1313 --disableFastRender
```

浏览器打开：

```text
http://127.0.0.1:1313/
```

说明：

- 改完文件后，页面通常会自动刷新
- 如果浏览器还是旧内容，按 `Ctrl + F5` 强制刷新

## 7. 你当前的发布结构

你这个仓库现在有两个重要分支：

### `main`

放 Hugo 源码。

### `master`

放构建后的静态页面。

GitHub Pages 直接从 `master` 分支根目录出站。

所以逻辑是：

- 改源码 -> 推 `main`
- 发布静态页 -> 推 `master`

---

## 8. 一键发布脚本

你现在已经有一个一键发布脚本：

```text
Z:\my_blogs\publish.ps1
```

最常用命令：

```powershell
cd "Z:\my_blogs"
.\publish.ps1
```

它会自动做这些事：

1. 检查当前是不是在 `main`
2. 检查工作区是否干净
3. 把源码分支推到 `origin/main`
4. 运行 `hugo --minify`
5. 复制 `public`
6. 把静态文件强制推到 `origin/master`

### 如果你只想发布静态文件，不重复推源码

```powershell
.\publish.ps1 -SkipSourcePush
```

### 如果你想改发布提交信息

```powershell
.\publish.ps1 -CommitMessage "deploy: update homepage"
```

## 10. GitHub Actions 自动化

项目里还有一套自动发布工作流：

```text
Z:\my_blogs\.github\workflows\hugo.yml
```

它的作用是：

- 当你 push 到 `main`
- GitHub 自动安装 Hugo
- 自动构建 `public`
- 自动把静态文件同步到 `master`

也就是说，将来如果这个 workflow 正常运行，你甚至只执行：

```powershell
git add .
git commit -m "update: xxx"
git push origin main
```

剩下的构建和发布 GitHub 会自动做。

## 11. GitHub Pages 检查方法

仓库地址：

```text
https://github.com/luxuryzcx/luxuryzcx.github.io
```

站点地址：

```text
https://luxuryzcx.github.io/
```

检查步骤：

1. 打开 GitHub 仓库
2. 进入 `Settings`
3. 点 `Pages`
4. 确认来源是：
   - Branch: `master`
   - Folder: `/ (root)`

如果刚推完看不到：

- 等 1 到 3 分钟
- 按 `Ctrl + F5`
- 看 `Deployments` 或 `Pages` 状态是否成功

## 12. 常见问题

### 本地页面没变化

先强刷：

```text
Ctrl + F5
```

### Hugo 命令不能用

检查：

```powershell
hugo version
```

如果找不到命令，重开一个终端再试。

### Git push 提示认证失败

说明 GitHub 凭证有问题，需要重新登录或者换 PAT。

### 线上页面样式错乱

大概率是：

- `baseURL` 配错
- `master` 没推成功

你当前正确的 `baseURL` 是：

```toml
baseURL = "https://luxuryzcx.github.io/"
```
