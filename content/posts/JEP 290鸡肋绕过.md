## 前言

由于很多框架是 Java 写的，所以研究 Java 安全刻不容缓。这篇文章收录在「Java 安全 持续学习」系列中，用来沉淀对应主题的分析与记录。

#### 绕过思路总结 (实战中的攻防对抗)

|**绕过场景**|**原理分析**|**应对建议**|
|---|---|---|
|**白名单过宽**|为了兼容业务，开放了 `java.*` 下的所有类，导致内置的高危类库被误放。|必须实施“强白名单”，仅允许确定业务需要的类。|
|**逻辑嵌套绕过**|利用白名单内的类触发序列化流中嵌套的恶意类，如果 Filter 不够严格（未校验嵌套深度）。|限制 `maxdepth` 和 `maxrefs`。|
|**配置错误**|在某些环境中，JEP 290 未被正确加载或被第三方框架覆盖。|检查 JVM 启动参数是否成功生效，确保过滤器位于调用链路的最前端。|
