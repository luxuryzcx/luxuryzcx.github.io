## 1. T3 协议 (WebLogic Proprietary Protocol)

- **定义**：WebLogic 专有的二进制通信协议，用于服务器间及客户端到服务器的通信。
    
- **默认端口**：7001。
    
- **安全本质**：
    
    - **反序列化入口**：该协议支持 Java 原生序列化，是 WebLogic 反序列化漏洞最主要的利用通道。
        
    - **绕过能力**：基于 TCP 二进制传输，由于不经过 HTTP 层，传统的 WAF 很难对其内部携带的序列化对象进行深度包检测（DPI）。
        
- **防御重点**：禁用 T3 协议、限制 7001 端口访问、启用 JEP 290 序列化过滤器。
    

## 2. IIOP 协议 (Internet Inter-ORB Protocol)

- **定义**：基于 CORBA 标准的分布式对象通信协议，用于跨编程语言的通信。
    
- **默认端口**：7001 或 7002。
    
- **安全本质**：
    
    - **更强的隐蔽性**：许多安全防御策略和扫描器仅针对 T3 协议优化，IIOP 往往成为“防御真空地带”。
        
    - **攻击载体**：同样支持 Java 序列化传输，攻击者可利用 IIOP 触发远程方法调用（RMI）或 EJB 执行，从而到达反序列化 Sink 点。
        
- **防御重点**：如果业务无需跨语言分布式调用，建议在 WebLogic 配置中彻底禁用 IIOP。
    

## 3. 核心对比总结

|**特性**|**T3 协议**|**IIOP 协议**|
|---|---|---|
|**协议性质**|WebLogic 私有二进制协议|CORBA 标准通信协议|
|**适用场景**|WebLogic 内部通信、Java 客户端|分布式系统、跨语言互操作|
|**安全态势**|重点防御对象，WAF 覆盖较多|较常被忽视，隐蔽性极高|
|**漏洞利用**|绕过 HTTP 防御，直接投递序列化流|利用 RMI/EJB 调用链触发反序列化|

## 4. 安全建议 (加固清单)

1. **协议禁用**：若无必要，在 WebLogic 控制台中禁用 T3 和 IIOP 通道。
    
2. **网络隔离**：限制 7001/7002 端口仅允许内网特定 IP 访问。
    
3. **过滤器增强**：必须在 JVM 启动参数中强制配置 `jdk.serialFilter` (JEP 290) 以实施白名单防御。
    
4. **RASP 监控**：在 JVM 层面通过 RASP 挂钩 `ObjectInputStream.readObject()`，审计所有流经这些协议的序列化对象。
